# Observer–Target Tracking Alignment — Cross-Correlation Pipeline

Demo implementation of the full analysis pipeline from the project brief
(*Continuous Time-Series Cross-Correlation & Statistics*), running on
simulated data with known ground truth. The client's real tracking files
drop into `data/raw/` with only the ingest cell adapted.

The simulation is grounded in the continuous-annotation literature: observer
reaction lags of 1–6 s (Mariooryad & Busso, 2015; Khorram et al., 2019) and
smooth, gain/bias-varying joystick traces as in the CASE dataset (Sharma et
al., 2019, *Scientific Data*). Because each simulated observer carries a
hidden true lag, the notebook verifies the pipeline recovers the injected
ground truth (100% within ±1 s) — a correctness check real data cannot offer.
Three figures illustrate the traces, one dyad's 16-step cross-correlation
function, and the group-mean CCFs.

## What the pipeline does

1. **Tier 1** — zero-lag Pearson *r* for all 168 observer–target sessions (28 observers × 6 trials)
2. **Tier 2** — time-lagged cross-correlation at every 1000 ms step from −5 s to +10 s
   (exactly 16 lags/session = 2,688 correlations), boundary rows **dynamically omitted**
   (never zero-filled); extracts **Peak r** (max absolute correlation) and **Peak lag**
3. **Tier 3** — Mixed-Design ANOVA (Group × Target) + complementary linear
   mixed-effects model, Test vs. Control, reporting *F*, *df*, exact *p*

## Reproduce

```bash
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/jupyter nbconvert --to notebook --execute --inplace tracking_crosscorr_analysis.ipynb
```

Fixed seed (`RNG_SEED = 42`) → every rerun is bit-identical; the notebook prints the
MD5 of the Master Metrics Sheet as a verification checksum. Running the mirror
`.py` regenerates all output files with identical hashes (verified).
